# Plan: Landing page CalendarAI con navbar y menú hamburguesa

## Objective
Reemplazar el scaffold por defecto de Next.js con una landing page profesional para **CalendarAI**, una app de gestión de calendario, incluyendo un navbar responsivo con menú hamburguesa y un link de Login.

## Current State
- `app/page.tsx:1-66` — scaffold genérico de `create-next-app`, logos de Next.js/Vercel, sin relación con el producto
- `app/layout.tsx:15-18` — metadata por defecto: `title: "Create Next App"`, `description: "Generated by create next app"`
- `app/layout.tsx:28-31` — body renderiza `{children}` directamente, sin navbar global
- `src/components/` — **no existe**; será el primer componente del proyecto
- No existe `app/login/` ni ninguna ruta adicional

## Assumptions
- **Paleta de colores**: azul/índigo (`blue-600`, `indigo-600`, `indigo-700`, `indigo-50`) — consistent con el branding de CalendarAI
- **Navbar en `layout.tsx`**: se incluye globalmente para que todas las rutas futuras lo hereden
- **Link Login**: `<Link href="/login">` de Next.js, apunta a `/login` sin funcionalidad (la ruta no existe aún — se implementará en el futuro)
- **Sin logo SVG personalizado**: se usa texto "CalendarAI" con ícono de calendario inline (SVG simple) hasta tener assets definitivos
- **Dark mode**: se mantiene el soporte existente (`dark:` prefix) con variantes oscuras de la paleta índigo
- **Contenido landing**: hero section con headline + subtítulo + CTA, sección de features (3 cards) y footer mínimo

## Edge Cases to Handle
- **Link `/login` sin ruta**: el link apuntará a `/login` que dará 404 — es comportamiento esperado y aceptado en este plan
- **Menú hamburguesa abierto en resize**: si el usuario expande la ventana con el menú abierto, el menú desktop debe ser visible y el menú móvil ocultarse (`hidden md:flex`) sin estado inconsistente
- **Fuentes Geist**: ya están configuradas globalmente; `font-sans` en Tailwind las aplica automáticamente — no se necesita configuración adicional

## Out of Scope
- Funcionalidad de Login/autenticación (se evaluará en un plan futuro)
- Animaciones o transiciones complejas
- Internacionalización (i18n)
- Tests automatizados de UI
- Logo SVG personalizado definitivo de CalendarAI
- Página `/login` o cualquier ruta adicional
